# 銀行コードレビューチェックリスト

## CRITICAL — リリースブロッカー（必ず修正）

### C1. トランザクション安全性

- [ ] **@Transactional の存在**: すべての書き込み操作に `@Transactional` が適用されている
- [ ] **rollbackFor の指定**: `@Transactional(rollbackFor = Exception.class)` で checked exception もロールバック対象
- [ ] **プロキシ迂回の回避**: 同一クラス内の `@Transactional` メソッドを直接呼び出していない（自己呼び出しの罠）
- [ ] **トランザクション内での外部呼び出し禁止**: HTTP呼び出しやメッセージ送信がトランザクション内で行われていない
- [ ] **readOnly の適切な使用**: 参照のみの操作には `@Transactional(readOnly = true)` が設定されている
- [ ] **Propagation の適切な選択**: REQUIRED / REQUIRES_NEW の使い分けが正しい

### C2. 金銭計算

- [ ] **BigDecimal 使用**: 金額に `double` / `float` を使用していない
- [ ] **文字列コンストラクタ**: `new BigDecimal("0.1")` を使用（`new BigDecimal(0.1)` は禁止）
- [ ] **RoundingMode 明示**: 除算時に `RoundingMode` が指定されている
- [ ] **比較方法**: `compareTo()` を使用（`equals()` はscaleが異なると false）
- [ ] **金額バリデーション**: 負数・ゼロ・上限値のバリデーションがある
- [ ] **通貨コード**: 複数通貨がある場合、通貨コードが考慮されている

### C3. 監査ログ

- [ ] **状態変更の記録**: すべての CREATE / UPDATE / DELETE 操作に監査ログがある
- [ ] **独立トランザクション**: 監査ログが `Propagation.REQUIRES_NEW` で記録される
- [ ] **PII 除外**: 監査ログに個人情報（口座番号全桁、氏名、電話番号等）が含まれていない
- [ ] **失敗の記録**: 操作が失敗した場合も監査ログが記録される
- [ ] **5W1H**: 誰が、何を、いつ、どこで、どのように、結果が記録されている

## HIGH — マージブロッカー（原則修正）

### H1. 排他制御

- [ ] **@Version フィールド**: 更新対象のエンティティに `@Version` がある（楽観ロックの場合）
- [ ] **OptimisticLockingFailureException のハンドリング**: キャッチしてリトライまたはエラーレスポンスを返している
- [ ] **ロック順序の統一**: 複数レコードをロックする場合、ID昇順等で統一している（デッドロック防止）
- [ ] **ロックタイムアウト**: 悲観ロック使用時にタイムアウトが設定されている
- [ ] **ロック範囲の最小化**: ロック保持時間が最小限に抑えられている

### H2. 冪等性

- [ ] **冪等キー**: POST / PATCH リクエストに冪等キーによる重複チェックがある
- [ ] **重複検出の応答**: 重複リクエスト時に初回と同じレスポンスを返している
- [ ] **ユニーク制約**: データベースレベルでの重複防止制約がある
- [ ] **有効期限**: 冪等キーの有効期限が設定されている

### H3. 入力バリデーション

- [ ] **Null チェック**: 必須パラメータの null チェックがある
- [ ] **範囲チェック**: 数値パラメータに最小値・最大値の制限がある
- [ ] **フォーマットチェック**: 口座番号等のフォーマットバリデーションがある
- [ ] **SQLインジェクション対策**: パラメータ化クエリを使用している
- [ ] **XSS対策**: ユーザー入力を出力する際にエスケープしている

### H4. エラーハンドリング

- [ ] **グローバル例外ハンドラー**: `@ControllerAdvice` でグローバルな例外処理がある
- [ ] **内部情報の非漏洩**: エラーレスポンスにスタックトレースや内部クラス名が含まれていない
- [ ] **適切なHTTPステータス**: 400, 404, 409, 500 等が適切に使い分けられている
- [ ] **リトライ可能性の明示**: レスポンスヘッダーまたはボディでリトライ可否が示されている

## MEDIUM — 改善推奨

### M1. データ保護

- [ ] **PII暗号化**: 個人情報がデータベースで暗号化されている
- [ ] **ログのPII除外**: アプリケーションログに個人情報が出力されていない
- [ ] **TLS通信**: 外部通信がTLSで暗号化されている

### M2. パフォーマンス

- [ ] **N+1クエリ**: `@EntityGraph` や `JOIN FETCH` で N+1 問題を回避している
- [ ] **ページネーション**: 大量データの取得に `Pageable` を使用している
- [ ] **インデックス**: 検索条件に使用されるカラムにインデックスがある
- [ ] **コネクションプール**: データベース接続にコネクションプールが設定されている

### M3. コード品質

- [ ] **関数の長さ**: 50行以下
- [ ] **ファイルの長さ**: 800行以下
- [ ] **ネストの深さ**: 4階層以下
- [ ] **マジックナンバー**: 定数化されている
- [ ] **テストカバレッジ**: 80%以上
