# AI自己レビュー チェックリスト — ミッションクリティカルシステム向け

`git diff` の差分に対して、以下のチェックリストを適用する。
該当しない項目はスキップし、該当する項目のみ検証する。

---

## CRITICAL — 1つでも違反があれば修正必須

### C1. トランザクション安全性
- [ ] すべての書き込み操作にトランザクション宣言が適用されている
- [ ] 例外発生時のロールバック設定が適切（チェック例外も含む）
- [ ] 自己呼び出しによるプロキシ迂回がない
- [ ] トランザクション内で外部API呼び出しをしていない
- [ ] 複数テーブル更新時に部分コミットが発生しない

### C2. 金銭計算の安全性
- [ ] 高精度小数型（Decimal / BigDecimal）を使用している（double/float 禁止）
- [ ] 文字列コンストラクタを使用している（`Decimal("0.1")` — `Decimal(0.1)` は禁止）
- [ ] 丸めモード（RoundingMode）が明示的に指定されている
- [ ] ゼロ・負数・上限のバリデーションがある
- [ ] 通貨コードが考慮されている（異なる通貨間の演算を防止）

### C3. 監査ログ
- [ ] すべての状態変更操作に監査ログがある
- [ ] 監査ログは独立トランザクション（REQUIRES_NEW）で記録される
- [ ] 処理失敗時も監査ログが記録される
- [ ] 監査ログにPII（個人識別情報）が平文で含まれていない
- [ ] ハッシュチェーンによる改ざん防止が維持されている

---

## HIGH — マージ前に修正すべき

### H1. 排他制御
- [ ] 残高更新に楽観ロックまたは悲観ロックが実装されている
- [ ] 楽観ロック例外のリトライハンドリングがある
- [ ] 複数リソースのロック時、ロック順序が統一されている（デッドロック防止）
- [ ] ロックタイムアウトが設定されている

### H2. 冪等性
- [ ] POST/PATCHリクエストに冪等キーが実装されている
- [ ] 同一冪等キーによる重複実行が防止されている
- [ ] リトライ時に副作用（二重振込等）が発生しない
- [ ] 冪等キーの有効期限管理がある

### H3. 入力バリデーション
- [ ] すべてのユーザー入力がバリデーションされている
- [ ] SQLインジェクション対策（パラメータ化クエリ）
- [ ] XSS対策（出力エスケープ）
- [ ] リクエストサイズ制限がある

### H4. エラーハンドリング
- [ ] 例外がキャッチされず漏洩していない
- [ ] エラーレスポンスにスタックトレースや内部情報が含まれていない
- [ ] 適切なHTTPステータスコードが使用されている
- [ ] ビジネス例外と技術的例外が区別されている

---

## MEDIUM — 可能であれば対応

### M1. データ保護
- [ ] PII（個人識別情報）が暗号化されている
- [ ] ログ出力にPIIが含まれていない
- [ ] データベース接続にTLSが使用されている
- [ ] 不要なデータが保持されていない（データ最小化）

### M2. セキュリティ
- [ ] シークレットがハードコードされていない（環境変数またはシークレットマネージャー使用）
- [ ] CORS設定が適切（ワイルドカード禁止）
- [ ] CSRFトークンが実装されている
- [ ] セキュリティヘッダーが設定されている（X-Content-Type-Options等）

### M3. コード品質
- [ ] 関数が小さい（50行未満）
- [ ] ファイルが適切な範囲（800行未満）
- [ ] 深いネスト（4階層以上）がない
- [ ] イミュータブルパターンが使用されている（既存オブジェクトの変更なし）
- [ ] ハードコードされた値がない（定数またはconfigを使用）
