# adflow — ADR駆動 AI開発ワークフロー

> adflow プラグインが有効です。以下のコマンドでADR駆動の開発ワークフローを実行できます。

## アクティブなワークフロー状態
!`ls docs/*/.adflow-context.md 2>/dev/null`

## コマンド一覧

| コマンド | 説明 | いつ使う |
|---------|------|---------|
| `/workflow [機能名]` | 全フェーズを順番に実行する統合コマンド | 新機能の開発を最初から最後まで進める時 |
| `/adr [タイトル]` | ADR（アーキテクチャ決定記録）を作成 | 設計判断を記録・検討する時 |
| `/spec [ADR番号]` | 仕様書を作成（Mermaid図・API仕様・データモデル） | ADR承認後に詳細設計を行う時 |
| `/stack-plan [機能名]` | スタックPR実装計画を作成 | 仕様書承認後にPR分割・タスク定義を行う時 |
| `/stack-loop [機能名]` | 実装ループ（TDD→レビュー→PR作成）を実行 | 計画承認後に実装を開始する時 |

## ワークフロー進行

```
/adr → /spec → /stack-plan → /stack-loop
  ↓       ↓         ↓            ↓
 ADR    仕様書   PR計画書     実装(TDD/Review/PR)
```

各ステージの成果物は `docs/{NNNN-機能名}/` に保存され、コンテキストファイル（`.adflow-context.md`）が状態を管理します。`/clear` しても途中から再開できます。

## キーワード検出ルール

以下のキーワードを検出した場合、対応するコマンドの使用を提案してください:

| キーワード | 推奨コマンド |
|-----------|------------|
| 「新機能」「実装したい」「機能追加」「作りたい」 | `/workflow` |
| 「設計判断」「アーキテクチャ」「技術選定」「ADR」 | `/adr` |
| 「仕様書」「設計書」「API設計」「データモデル」 | `/spec` |
| 「PR計画」「タスク分割」「実装計画」 | `/stack-plan` |
| 「実装開始」「コーディング」「TDD」 | `/stack-loop` |

## 次のアクション

アクティブなワークフローがある場合、上記の状態表示から次に実行すべきコマンドを案内してください。ワークフローがない場合、`/workflow [機能名]` で新しいワークフローを開始することを提案してください。

## ドメイン品質チェック（ビルトイン）

adflow はミッションクリティカルシステム向けの品質チェックをビルトインで提供:
- 金額計算: 高精度小数型（Decimal/BigDecimal）必須
- 監査ログ: すべての状態変更に記録
- トランザクション: 境界の明示的設計
- 排他制御: 楽観ロック / 悲観ロック
- 冪等性: リトライ安全な設計
- PII保護: マスキング・暗号化
