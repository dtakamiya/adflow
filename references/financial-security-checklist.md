# 金融セキュリティチェックリスト

## CRITICAL — リリース前に必ず確認

### C1. 金銭計算の安全性
- [ ] `BigDecimal` を使用（`double`/`float` 禁止）
- [ ] `RoundingMode` が明示的に指定されている
- [ ] 通貨コードが考慮されている
- [ ] オーバーフロー/アンダーフローの検証

### C2. トランザクション安全性
- [ ] すべての書き込み操作に `@Transactional` が適用されている
- [ ] `rollbackFor` が適切に設定されている
- [ ] 自己呼び出しによるプロキシ迂回がない
- [ ] トランザクション内で外部API呼び出しをしていない

### C3. 認証・認可
- [ ] すべてのエンドポイントに認証が必要
- [ ] 権限チェックがビジネスロジック層で実装されている
- [ ] IDOR（Insecure Direct Object Reference）対策済み
- [ ] セッション管理が適切

### C4. 監査ログ
- [ ] すべての状態変更操作に監査ログがある
- [ ] 監査ログは独立トランザクション（REQUIRES_NEW）
- [ ] 監査ログにPIIが含まれていない
- [ ] 改ざん防止策が実装されている

## HIGH — マージ前に確認

### H1. 排他制御
- [ ] 残高更新に楽観ロック or 悲観ロックが実装されている
- [ ] デッドロック防止策（ロック順序統一）がある
- [ ] `OptimisticLockingFailureException` のハンドリング
- [ ] ロックタイムアウトが設定されている

### H2. 冪等性
- [ ] POST/PATCHリクエストに冪等キーがある
- [ ] 重複リクエストの検出が実装されている
- [ ] リトライ時に副作用が発生しない
- [ ] 冪等キーの有効期限管理がある

### H3. 入力バリデーション
- [ ] すべてのユーザー入力がバリデーションされている
- [ ] SQLインジェクション対策（パラメータ化クエリ）
- [ ] XSS対策（出力エスケープ）
- [ ] リクエストサイズ制限

### H4. エラーハンドリング
- [ ] 例外がキャッチされず漏洩していない
- [ ] エラーレスポンスに内部情報が含まれていない
- [ ] スタックトレースがクライアントに返されていない
- [ ] 適切なHTTPステータスコードが使用されている

## MEDIUM — 定期レビューで確認

### M1. データ保護
- [ ] PII（個人識別情報）が暗号化されている
- [ ] ログにPIIが出力されていない
- [ ] データベース接続にTLSが使用されている
- [ ] 不要なデータが保持されていない（データ最小化）

### M2. パフォーマンス
- [ ] N+1クエリが発生していない
- [ ] 適切なインデックスが作成されている
- [ ] コネクションプールが設定されている
- [ ] レート制限が実装されている

### M3. 依存関係セキュリティ
- [ ] 依存ライブラリに既知の脆弱性がない
- [ ] セキュリティアップデートが適用されている
- [ ] 不要な依存関係が削除されている

### M4. API セキュリティ
- [ ] CORS設定が適切
- [ ] CSRFトークンが実装されている
- [ ] レスポンスヘッダーが適切（X-Content-Type-Options, etc.）
- [ ] APIバージョニングが実装されている
